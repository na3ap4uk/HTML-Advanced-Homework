"autocomplete"

Атрибут дозволяє заповнити форму автоматично, якщо необхідний текст був введений раніше. Сам текс зберігається браузером, ним же можна і відключити дію атрибуту. Можна задати значення - yes , no
Доступний для усіх браузерів.

"autofocus"

Даний атрибут автоматично встаовлює фокус в полі форми(<input autofocus>). Можна без наведення курсору відразу вводити текст у форму. 
Не доступний для IE.

"multiple"

Атрибут дозволяє зазначати, вводити(напр. пошту), або обирати (напр. в дроплісті) одночасно декілька значень. Можна ввести числове значення для multiple, яке обмежить к-сть можливого вводу.
Доступний для Chrome, IE, Firefox (4.0)


"placeholder"

Атрибут для форми, зазначає текст, який міститиметься в полі форми, текст стилізований і зникне у разі фокусу а нього. 
Доступний для усіх браузерів.

"pattern (regexp)"

Даний атрибут зазначає смислове значення для тексту, який має бути введений. У разі неправильного заповнення форми, сама форма не надсилатиметься. 
regexp - алгоритм для введення значень. Дуже зручний, напр., для паролів, напр.: введене значення[A-Za-z] - означатиме лише латиницю для введення.
Не доступний для Safari

"readonly"

Даний атрибут, якщо додається до <input> не дозволяє користувачу змінювати текстове поле, але за допомогою скріптів його вміст можна буде змінювати.
Доступний для усіх браузерів.

"disabled"

Блокує доступ до поля форми,яке буде відображатися недоступним. За допомогою скріптів можна змінити таке блокування.
Доступний для усіх браузерів.

"range"

Т.зв. "повзунок". За допомогою даного елементу можем задавати числове значення тільки визначеного в коду. Можливі варіанти повзунком. Синтаксис буде, напр., таким: <input type="range" min="0" max="100" step="10" value="20">.
Можемо для цього елементу прописати скріпт ззовні і прив'язати
за допомогою "oninput"

"tel"
Текстове поле за допомогою даного типу означатиме номер телефону, перевірятись на правильність синтаксису (як, напр. в мейлі) дане поле не буде, але ми можемо задати елемент pattern із необхідними значеннями.

"time"
Даний тип позначатиме час, можемо задати мінімальні і максимальні значення, тобто "від" і "до".
 
"url" 

Даний елемент позначає адресу до файлу, сторінки чи документу.

"number"

Елемент означає введення чисел. Не підтримується Firefox. 

<dl> <dt> <dd>

Є тегами, що позначають список. Даний список не буде нумерований чи стилізованим (крапкою, рискою). Список буде мати текстовий характер і використовується в основному для визначення термінів. <dl> позначатиме початок тексту, <dt> його заголовки(терміни), <dd> міститиме відповідний текст. Такі списки є зручними для стилізації. 
Теги підтримуються усіма браузерами.
 
<fieldset> 

Даний тег призначений для групування елементів форми. Виглядає в браузері як рамка. Дане групування полегшує роботу з формами (групує напр. логін і пароль, і дає запит).
Доступний для усіх браузерів.

<legend> 
Тег є заголовком для групи елементів форми fieldset. Відображається вбудованим в рамку.
Доступний для усіх браузерів.

<datalist>

Тег використовується для створення варіантів для введеного тексту. Текст вводиться у поле і можливі варіанти виводяться внизу під полем. Варанти зазначаємо в option.
Тег не підтримується Safari, Android i iOS.

<optgroup> 

Тег служить загаловком в списку елементів <option>. Стилізується жирним шрифтом. Обрати його значення у списку не можна. Зміщує елементи <option> вправо.
Доступний для усіх браузерів.

<progress> 

Тег графічно показує прогрес, як, напр., при завантаженні, встановленні. Зміна значення здійснюється через JavaScript.
Тег не підтримується Android i iOS.

<meter> 

Тег графічно (як і в <progress>) демонструє якесь числове значення, містить атрибути value, max, min, high, optimum.
Тег не підтримується IE, Android i iOS.

<iframe> 

Тег створює плаваючий фрейм, в якому знаходиться зазвичай елемент із зовнішнього інтернет ресурсу.
Доступний для усіх браузерів.

<embed>
Використовується для загрузки і відображення об'єктів, які браузер попередньо не сприймає. Для цих об'єктів є необхідними плагіни, або допоміжні програми. Тег не потребує закриваючого тега.
Підтримується усіма браузерами. Тег може містити атрибут pluginspage у якому буде міститись адреса сторінки, звідки можна буде закачати і встановити плагін.

<object>

Даний тег повідомляє браузеру як загружати і відображати об'єкт, який браузер первинно не розуміє. Як правило ці об'єкти підключаються до плагіна. В даний тег вкладають інші теги, які дозволять ігнорувати плагін і відобразити об'єкт.
Часто містить тег <param>, який передає додаткові параметри для відображення об'єкта.
